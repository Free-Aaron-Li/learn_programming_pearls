# 第1章 开篇

对于开篇所提到的对电话号码进行排序，我首先想到的是`位排序`。

首先要求：

- 每条记录都是7位`整数`
- 只有1MB内存可用
- 号码不能重复
- 排序允许执行时间理想情况为10s

采用位排序可以解决：

- 内存较小问题，毕竟一个电话号码占20位（百万级别）,1MB可以存储2^24位（千万级）
- 位排序决定了不存在号码重复情况

上述问题在排序上有三个不常见的属性：

- 输入数据限制在相对较小的范围内（在10^7的大小内）
- 数据没有重复
- 对于每条数据，除了单一整数外，没有任何其他关联数据

所以，可以使用一个具有1000万个位的字符串表示这个文件，使用20位长的字符串表示每个电话号码元素。

**对小问题的仔细分析有时可以得到明显的实际益处**。
